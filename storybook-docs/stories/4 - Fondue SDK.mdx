# Fondue SDK

> **This is a work in progress. The API is not yet stable and will change.**

A Node API for machine-readable access to the Fondue design system as a navigable graph. Designed for MCP servers, CLI generators, documentation pipelines, AI agents, and any other tooling that needs to introspect Fondue programmatically.

## Install

```sh
pnpm add @frontify/fondue
```

## 30-second tour

```ts
import { components, tokens } from '@frontify/fondue/sdk';

// Look up by id
const button = components.get('Button');

// Read scalar data
button?.props.length;                                 // 14
button?.examples.find((e) => e.isCanonical)?.code;    // "<Button …>…"

// Walk the graph
button?.category().name;                              // 'input'
button?.related().map((c) => c.name);                 // ['SplitButton', 'Link']

// Query
components.where({ category: 'input', status: 'released' });
components.tag('cta')?.list();
tokens.where({ category: 'colors', themeable: true });

// Icons live in the components graph under category: 'icon'
components.where({ category: 'icon' });               // all icons
components.get('IconAdobeCreativeCloud');             // { category: 'icon', … }
```

## Common query shape

`components`, `tokens`, `tokens.utilities`, and every facet node implement
the same five members:

<table>
    <thead>
        <tr>
            <th>Member</th>
            <th>Returns</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>
                <code>list()</code>
            </td>
            <td>All nodes, in stable order</td>
        </tr>
        <tr>
            <td>
                <code>get(id)</code>
            </td>
            <td>
                A node, or <code>undefined</code>
            </td>
        </tr>
        <tr>
            <td>
                <code>has(id)</code>
            </td>
            <td>Boolean</td>
        </tr>
        <tr>
            <td>
                <code>where(filter)</code>
            </td>
            <td>Matching nodes (filters AND-combine)</td>
        </tr>
        <tr>
            <td>
                <code>size</code>
            </td>
            <td>Total node count</td>
        </tr>
    </tbody>
</table>
