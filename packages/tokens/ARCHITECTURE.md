# Fondue Design Tokens — Architecture

This document explains how `@frontify/fondue-tokens` extracts design tokens from Figma and transforms them into CSS variables, a Tailwind config, and JSON outputs.

## Data Flow

```
Figma API (source of truth)
    │
    ▼
loadFigmaVariables.ts ─────► .tmp/tokens/all-tokens.json
    │
    ▼
buildStyleDictionary.ts (Style Dictionary v4)
    ├─► dist/css/base.css              (CSS custom properties)
    ├─► dist/tailwind/tailwind.config.js (Tailwind preset)
    ├─► .tmp/themes/{base,light,dark}.css
    └─► .tmp/objects/{light,dark,semantic,utilities}.json
    │
    ▼
buildTokens.ts (final assembly)
    ├─► dist/themes/themes.module.css   (merged theme CSS)
    ├─► dist/json/*.json                (merged JSON exports)
    └─► dist/legacy/                    (deprecated token compat)
```

## Build Process

Run `npm run build` which executes `tsx src/buildTokens.ts`.

### Step 1: Fetch Tokens from Figma

`src/loadFigmaVariables.ts` calls the Figma Variables API using a `FIGMA_ACCESS_TOKEN` env var and the file key from `config.json`.

The Figma file contains four types of token collections configured in `config.json`:

| Token Type | Contents |
|---|---|
| `base` | Primitives — colors, spacing, border-radius, border-width, font families |
| `theme` | Theme-specific colors (light/dark modes) |
| `semantic` | Non-color semantic tokens — shadows |
| `utility` | Typography utility compositions (body, heading text styles) |

The function resolves Figma variable aliases (cross-references between tokens), extracts mode-specific values (light/dark), and restructures the flat token list into a nested object. For example, `Color/Primary/Default` becomes `{ Color: { Primary: { Default: { $value } } } }`.

Shadow tokens get special handling — sub-properties (position-x, position-y, blur, spread, color) are grouped into a single shadow object.

Output: `.tmp/tokens/all-tokens.json`

### Step 2: Transform with Style Dictionary

`src/styleDictionary/buildStyleDictionary.ts` registers custom transforms and generates four output platforms from the intermediate JSON.

#### Custom Transforms

| Transform | What it does |
|---|---|
| `figma/colorToScaledRgbaString` | Converts `{r,g,b,a}` (0–1 floats) to `rgba(R,G,B,A)` |
| `figma/shadowToMatrix` | Converts shadow objects to CSS `Xpx Ypx Blurpx Spreadpx var(--ref)` |
| `number/roundToTwoDecimals` | Rounds numeric values to 2 decimal places |
| `value/convertPxToRem` | Converts px to rem (÷16) for font-size, line-height, border-radius, spacing |
| `value/convertValueToPx` | Keeps px unit for border-width, letter-spacing |
| `value/refToCSSVariable` | Converts `ref_token-name` → `var(--token-name)` |
| `name/kebabWithoutThemeName` | Strips theme name from token path |
| `fondue/transformFontFamily` | Adds fallback stack to font-family tokens |

#### Output Platforms

**cssBase** → `dist/css/base.css`
CSS custom properties under `:root`. Contains all non-theme tokens plus the default (light) theme values.

**cssThemes** → `.tmp/themes/{base,light,dark}.css`
Per-theme CSS files with class selectors (`.light`, `.dark`, `.base`). Later merged into `dist/themes/themes.module.css`.

**objects** → `.tmp/objects/*.json`
Nested JSON objects per token category. Used to assemble the final `dist/json/` files.

**tailwind** → `dist/tailwind/tailwind.config.js`
Tailwind preset with theme values and plugins. Generated by the custom format in `src/styleDictionary/tailwind/`.

### Step 3: Tailwind Config Generation

The Tailwind format is assembled from two parts:

**`tailwindFormat.ts`** — Orchestrates the output, calling `buildTheme()` and `buildUtils()`.

**`buildTheme.ts`** — Maps tokens to Tailwind theme keys: `colors`, `fontSize`, `fontWeight`, `fontFamily`, `letterSpacing`, `lineHeight`, `borderRadius`, `borderWidth`, `boxShadow`, and `extend.spacing`. Adds Tailwind defaults like `inherit`, `current`, `transparent` to colors.

**`buildUtils.ts`** — Creates Tailwind plugin utility classes for composite typography (e.g. `.body-medium` sets font-family, size, weight, line-height, letter-spacing, and text-transform in one class). Also includes the `ring-focus` utility from `src/static/customUtilities.ts`.

### Step 4: Final Assembly

`src/buildTokens.ts` merges the intermediate outputs:

1. Combines theme CSS files into `dist/themes/themes.module.css`
2. Merges JSON objects into `dist/json/all-tokens.json`, `colors.json`, `semantic.json`, `utilities.json`
3. Generates TypeScript `.d.ts` declaration files
4. Copies legacy compatibility files to `dist/legacy/`
5. Cleans up `.tmp/`

## Key Files

| File | Purpose |
|---|---|
| `config.json` | Figma file key, theme list, collection IDs, token type filters |
| `src/buildTokens.ts` | Main entry point — orchestrates the full pipeline |
| `src/loadFigmaVariables.ts` | Fetches and processes Figma API response |
| `src/styleDictionary/buildStyleDictionary.ts` | Style Dictionary config with custom transforms |
| `src/styleDictionary/cssFormat.ts` | Custom CSS output format (sorted variables) |
| `src/styleDictionary/tailwind/tailwindFormat.ts` | Tailwind config output format |
| `src/styleDictionary/tailwind/buildTheme.ts` | Maps tokens to Tailwind theme object |
| `src/styleDictionary/tailwind/buildUtils.ts` | Generates typography utility plugin classes |
| `src/static/customUtilities.ts` | Focus ring utility definition |
| `legacyTokens/` | Deprecated token mappings for backward compatibility |
| `tools/replaceLegacyTokens.ts` | CLI migration tool for old → new token names |


Legacy exports are available under `@frontify/fondue-tokens/legacy/*` for migration.

## Adding or Modifying Tokens

Tokens are managed in the Figma file — this package only reads them. To change tokens:

1. Update the token values in the Figma file
2. Set the `FIGMA_ACCESS_TOKEN` environment variable
3. Run `npm run build` to regenerate all outputs
4. If new token collections are added in Figma, update the collection IDs in `config.json`
